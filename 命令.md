# Rust 本地知识库搜索系统演示视频命令整理

## 一、录制前准备命令

进入项目目录：

```powershell
cd "E:\Desktop\Rust程序设计实训\Rust期末作业\rust-note-search"
```

提前编译项目，避免正式录制时等待时间过长：

```powershell
cargo build
```

清理旧搜索历史，避免 `history` 中出现以前的查询记录：

```powershell
Remove-Item examples/knowledge_base/.rns/history.json -ErrorAction SilentlyContinue
```

注意：上面这条清理历史的命令只是录制前准备，不需要在视频中展示。

---

## 二、正式录制命令顺序

### 1. 构建索引

```powershell
cargo run -- index examples/knowledge_base
```

---

### 2. 中文搜索

```powershell
cargo run -- search 所有权 --path examples/knowledge_base --top 2 --no-history
```

---

### 3. 英文搜索：默认 TF-IDF 排序

```powershell
cargo run -- search ownership --path examples/knowledge_base --top 3 --no-history
```

---

### 4. 英文搜索：切换为 BM25 排序

用于和上一个 `ownership` 的默认 TF-IDF 搜索结果进行对比。

```powershell
cargo run -- search ownership --path examples/knowledge_base --ranker bm25 --top 3 --no-history
```

---

### 5. 路径过滤前：全局搜索

先在整个知识库中搜索 `data`。

```powershell
cargo run -- search data --path examples/knowledge_base --top 5 --no-history
```

---

### 6. 路径过滤后：只保留 rust 路径下的结果

用于和上一个 `data` 的全局搜索结果进行对比。

```powershell
cargo run -- search data --path examples/knowledge_base --filter rust --top 5 --no-history
```

---

### 7. 查询语法和评分解释

这里的 `rust -trait` 表示搜索包含 `rust` 但排除 `trait` 的文档。

```powershell
cargo run -- search --path examples/knowledge_base --explain --no-history -- rust -trait
```

注意：这里最后的第二个 `--` 必须保留，因为 `-trait` 以减号开头，容易被命令行解析器误认为是参数。

---

### 8. 三种搜索方案对比

```powershell
cargo run -- compare data --path examples/knowledge_base --top 5
```

---

### 9. 写入一次搜索历史

前面的搜索命令大多使用了 `--no-history`，不会写入历史。这里运行一次普通搜索，用于后面展示历史记录。

```powershell
cargo run -- search data --path examples/knowledge_base --top 1
```

---

### 10. 查看搜索历史

```powershell
cargo run -- history --path examples/knowledge_base --limit 3
```
